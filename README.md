# SuperGuardian · 超级守护

> 面向 Windows 的进程守护与定时管理工具。用于监控指定程序、自动拉起异常退出的进程，并提供定时重启、定时运行、失败重试、邮件提醒、自我守护、托盘常驻、配置导入导出等完整能力。

---

## 项目简介

`SuperGuardian` 是一个以桌面界面为核心的 Windows 工具，适合用于：

- 常驻程序异常退出后自动恢复
- 需要定时重启的业务程序、采集程序、客户端程序
- 需要按固定计划自动启动的工具程序
- 需要图形化查看日志、快速调整参数、集中管理多个进程的场景

程序采用 **单实例运行** 设计：

- 如果已经有一个实例在运行，再次启动时不会重复打开新窗口
- 已运行实例会被自动唤起并显示到前台

## 主界面说明

主界面由两部分组成：

1. 顶部输入区
   - 输入程序路径或系统工具名称
   - 支持拖放文件
   - 支持附带启动参数
   - 支持 `.lnk` 快捷方式解析
2. 下方程序列表
   - 展示所有已添加项目

### 列表列说明

| 列名 | 说明 |
|------|------|
| 程序 | 显示程序名、图标；设置备注后显示备注文本 |
| 运行状态 | 如：运行中、未运行、未守护、已重启、定时运行 |
| 持续运行时长 | 目标进程当前已运行多久 |
| 上次重启(运行) | 最近一次守护重启、定时重启或定时运行时间 |
| 被守护次数 | 守护或定时运行触发的累计次数 |
| 持续守护时长 | 当前守护已持续多久 |
| 定时规则 | 当前生效的定时重启/定时运行规则摘要 |
| 下次重启(运行) | 最近一条即将触发的时间 |
| 启动延时 | 守护重启或定时重启前等待的秒数 |
| 操作 | 快速切换守护、定时重启、定时运行 |

### 列表交互

- 支持多选行操作
- 支持拖动行调整顺序
- 支持点击表头排序：**升序 → 降序 → 默认顺序**
- 支持右键表头显示/隐藏列
- 支持拖动表头调整列顺序
- `操作` 列始终固定在最右侧
- 支持保存列宽、列显隐和表头顺序
- 支持 `Delete` 删除选中项
- 支持 `F2` 快速编辑备注
- 双击非 `操作` 列可直接打开“设置启动程序/参数”对话框

## 快速开始

### 1. 添加程序

可通过以下方式添加：

- 点击 `选择程序`
- 将 `.exe` 或 `.lnk` 拖到输入框
- 手动输入完整路径
- 直接输入系统工具名称，例如 `cmd`、`powershell`、`notepad`
- 输入“路径 + 参数”，例如：
  - `C:\Windows\System32\notepad.exe`
  - `cmd /k`
  - `"C:\Program Files\App\app.exe" --mode test`

### 2. 开启守护

添加后，点击行内 `开始守护`：

- 如果目标进程未运行，会立即尝试启动
- 后续检测到进程退出时，会按设置自动重启
- 关闭守护后：
  - 持续守护时长清零
  - 被守护次数清零

### 3. 配置定时重启或定时运行

右键目标项后可配置：

- `定时重启规则`
- `定时运行规则`

两类规则都支持：

- **周期重复**：按天 / 小时 / 分钟设置间隔
- **固定时间**：指定时刻，可附带星期筛选

规则支持多条，且相同时间点只执行一次。

## 功能详解

## 1. 进程守护

守护是本程序的核心能力。

工作方式：

- 程序以固定定时器轮询目标进程状态
- 检测到目标进程退出后，按当前配置执行恢复
- 如配置了启动延时，会先等待指定时间再尝试启动
- 如启动失败，可进入失败重试流程

守护模式下会维护以下状态：

- 守护开始时间
- 最近一次守护重启时间
- 被守护次数
- 启动失败去重标记
- 重试状态

### 守护与其他模式的关系

- 守护 **可以与定时重启同时存在**
- 守护 **不能与定时运行同时生效**
- 当启用定时运行时，守护会自动关闭

---

## 2. 定时重启

定时重启会在计划时间到达时：

1. 终止目标进程
2. 如设置了启动延时，则等待指定秒数
3. 再次启动程序

### 规则类型

#### 周期重复

可组合：

- 天
- 小时
- 分钟

例如：

- 每 30 分钟
- 每 2 小时
- 每 1 天 6 小时

#### 固定时间

可设置：

- 每天固定某个时间
- 或指定星期几的固定时间

例如：

- 每天 `03:00`
- 每周一、三、五 `22:30`

### 特性

- 支持多条规则同时存在
- 自动计算下一次触发时间
- 避免过近时间内与守护重启重复干扰

---

## 3. 定时运行

定时运行与定时重启不同：

- 它**不会主动杀掉已有进程**
- 只会在规则触发时尝试启动一次目标程序

适合场景：

- 定时启动采集任务
- 定时运行某个工具程序
- 只关心定时拉起，不需要强制重启

### 特性

- 支持多条运行规则
- 启用后自动关闭守护与定时重启
- 可选择是否显示“持续运行时长”

如果未勾选 `监控持续运行时长`：

- 列表中的 `持续运行时长` 固定显示 `-`

---

## 4. 启动延时

可为单个程序设置启动延时。

用途：

- 避免目标程序本身带有重启逻辑时发生重复拉起
- 给程序更新、退出清理、资源释放预留缓冲时间

### 行为

- 适用于：守护重启、定时重启
- 不适用于：定时运行
- 可设范围：`0 ~ 86400` 秒
- `0` 表示关闭延时

---

## 5. 失败重试

当程序启动失败时，可进入自动重试流程。

### 可配置项

| 项目 | 说明 |
|------|------|
| 重试间隔 | 两次重试之间的等待秒数 |
| 最大重试次数 | 达到次数后停止，`0` 表示不限制 |
| 最大重试时长 | 达到时长后停止，`0` 表示不限制 |

### 行为

- 守护启动失败后可触发重试
- 定时重启失败后可触发重试
- 定时运行失败后可触发重试
- 重试成功后自动退出重试状态
- 重试耗尽后记录日志，并可发送邮件提醒

---

## 6. 邮件提醒

支持 SMTP 邮件通知。

### 两级开关

1. **全局开关**
   - 菜单栏 `选项 -> 邮件提醒`
   - 托盘菜单 `邮件提醒`
2. **单个程序开关**
   - 右键菜单 `邮件提醒设置`

### 可选择的提醒事件

- 守护触发重启
- 启动失败
- 定时重启失败
- 定时运行失败
- 进程退出
- 重试耗尽

### SMTP 配置项

- SMTP 服务器
- 端口
- TLS/SSL 开关
- 用户名
- 密码
- 发件人地址
- 发件人名称
- 收件人地址

并支持在配置界面中直接发送测试邮件。

### 去重机制

失败类通知会在下一次成功恢复前只提醒一次，避免重复轰炸。

## 7. 自我守护

程序支持自我守护模式。

开启后会使用当前可执行文件启动一个带 `--watchdog` 参数的看门狗进程，用来监控主进程：

- 主进程异常退出后，看门狗会尝试重新启动主程序
- 最多进行 5 次拉起尝试
- 失败时会逐步延长等待时间后继续尝试

### 细节

- 正常通过 `退出` 菜单退出时，会写入“手动退出”标记，看门狗不会误拉起
- 可通过 `测试 -> 测试自我守护` 强制结束主进程，验证恢复能力
- 看门狗相关事件记录在**运行日志**中

---

## 8. 开机自启

开启后会写入当前用户的 Windows 启动项。

特点：

- 不需要管理员权限
- 仅影响当前用户
- 关闭后会移除相应启动项

## 12. 重复添加白名单

默认情况下，同一路径程序不允许重复加入列表。

以下情况允许重复添加：

- 系统工具或可通过系统路径找到的内置工具
- 位于“允许重复添加的程序”白名单中的程序

### 白名单功能

- 支持手动添加程序
- 支持拖放 `.exe` / `.lnk`
- 对 `.lnk` 会自动解析真实目标路径
- 支持单独测试某个程序当前是否允许重复添加

### 日志分类

- 操作日志

## 16. 本地更新

支持从本地文件更新程序。

### 支持格式

- `SuperGuardian.exe`
- `.zip` 压缩包

### 更新流程

1. 选择新版本文件
2. 自动备份当前程序到 `bak/时间戳/`
3. 替换主程序文件
4. 保留最近 5 个备份版本
5. 自动重启软件

### ZIP 更新行为

- 先解压到临时目录
- 优先寻找 `SuperGuardian.exe`
- 若只存在唯一一个 `.exe`，也可作为更新目标

### 恢复旧版本

更新对话框支持从备份列表中恢复旧版本，并自动重启。

## 使用建议

- 对需要自动恢复的程序，优先开启“守护 + 合理启动延时”
- 对需要定期刷新运行环境的程序，使用“定时重启”
- 对只需要按计划拉起的程序，使用“定时运行”
- 对不稳定程序，建议同时配置“失败重试 + 邮件提醒”
- 在正式环境中，可开启“自我守护 + 开机自启”提升可恢复性
