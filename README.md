# SuperGuardian · 超级守护

> 轻量级 Windows 进程守护工具 —— 监控指定程序，异常退出后自动重启，并提供定时重启、定时运行、启动延时、失败重试、邮件提醒、自我守护等完整保障机制。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 🛡️ 进程守护 | 实时监控指定程序，退出后自动拉起 |
| 📌 列表置顶 | 任意程序可置顶显示，置顶项始终排在列表最前 |
| ⏰ 定时重启 | 支持任意数量规则（周期重复 / 固定时间），自动按计划重启程序 |
| 🕐 定时运行 | 与守护和定时重启互斥，按规则定时启动程序（不杀进程） |
| ⏳ 启动延时 | 程序退出后等待指定秒数再重启，避免与程序自身重启逻辑冲突 |
| 🔄 失败重试 | 启动失败后自动重试，可配置间隔、次数上限、时长上限 |
| 📧 邮件提醒 | 关键事件（启动失败、重试耗尽等）自动发送邮件通知，支持全局开关与每程序独立开关 |
| 🔁 自我守护 | 内置看门狗，SuperGuardian 自身崩溃后也能自动恢复 |
| 🚀 开机自启 | 一键写入 Windows 启动项，开机后自动运行 |
| 🎨 明暗主题 | 自动识别系统主题，支持手动切换浅色 / 暗色模式 |
| 📋 配置管理 | 配置可导入 / 导出 / 重置，便于迁移和备份 |
| 📜 运行日志 | 看门狗日志与操作日志分文件记录，随时可查 |
| 🔗 快捷方式 | 支持直接添加 `.lnk` 快捷方式，自动解析真实路径 |
| 🪟 托盘常驻 | 关闭窗口后最小化至系统托盘，不影响桌面 |
| 🔽 启动时最小化 | 可选启动时仅显示托盘图标，不弹出主界面 |
| 🔃 列排序 | 点击表头排序（升序→降序→默认循环），排序状态跨重启保留，数据变化时自动维持排序 |
| 👁️ 列显隐 | 右键表头可勾选显示/隐藏任意数据列（"操作"列除外） |
| ↔️ 列宽自适应 | 拖动列宽时"操作"列固定在右侧，其余列按比例重新分配 |
| 🔄 本地更新 | 选择新版本程序文件一键更新，自动备份旧版本（最多保留 5 个） |

## 界面预览

程序主界面包含一个程序列表和顶部输入栏。输入框和按钮高度紧凑，文字垂直居中。顶部输入框左侧与列表左侧对齐，最后一个按钮右侧与列表右侧对齐。列表包含以下 9 列：

| 列名 | 说明 |
|------|------|
| 程序 | 程序文件名及图标，设置备注后显示备注名称 |
| 运行状态 | 运行中 / 未运行 / 未守护 / 已重启 / 定时运行 |
| 持续运行时长 | 自开启守护起的累计运行时间 |
| 上次重启/运行 | 最近一次守护/定时重启/定时运行的时间戳 |
| 守护次数 | 守护触发重启的累计次数（关闭守护或软件重启时清零；定时运行时显示"-"） |
| 定时重启/运行规则 | 当前配置的定时重启或定时运行规则（支持多条） |
| 下次重启/运行 | 下一次定时重启或定时运行的预计时间 |
| 启动延时 | 触发守护/定时重启前的等待时长（秒） |
| 操作 | 开始守护/关闭守护 · 开启定时重启/关闭定时重启 · 开启定时运行/关闭定时运行 |

- 操作列的每个按钮显示完整名称，按钮大小根据文字自适应。
- 列表中文字显示不全时，鼠标悬停可查看完整内容。
- 列宽支持拖动自定义，可通过菜单**操作 → 重置列宽**还原默认比例，并自动保存。
- 拖动调整列宽时，"操作"列始终固定在最右侧，其余未调整的列按原有比例自动重新分配。
- 点击表头可排序（升序→降序→默认循环），排序状态跨软件重启保留。数据列内容变化时自动维持当前排序。
- 右键点击表头（"操作"列除外）可打开列显隐菜单，勾选/取消勾选以显示或隐藏列，设置跨重启保留。

## 快速上手

### 添加程序

有三种方式将程序加入守护列表：

1. **拖放**：直接将 `.exe` 或 `.lnk` 文件拖入顶部输入框
2. **浏览**：点击"选择程序"按钮，通过文件对话框选取
3. **手动输入**：在输入框中输入程序路径或名称，点击"添加"或按 **Enter** 键

**输入格式：**

- 完整路径：`C:\Windows\System32\notepad.exe`
- 带引号路径（路径含空格时）：`"C:\Program Files\app.exe"`
- 系统程序名称（不区分大小写）：`cmd`、`PowerShell`、`notepad`、`ping` 等，自动在系统 PATH 中查找完整路径
- 路径 + 启动参数：`cmd /k`、`notepad C:\test.txt`、`"C:\Program Files\app.exe" --arg1`

> 同一路径不会重复添加。所有输入框右键菜单已汉化，保留原始图标与快捷键样式（撤销、重做、剪切、复制、粘贴、删除、全选）。

### 开启守护

- 点击列表行末的 **"开始守护"** 按钮
- 若程序当前未运行，守护开启后将立即启动程序
- 关闭守护后，守护次数自动清零

> 💡 双击列表中的程序行可快速打开 **"设置启动程序/参数"** 对话框。

### 开启定时重启

- 点击列表行末的 **"开启定时重启"** 按钮，或右键选择 **"定时重启规则"**
- 支持添加多条规则，每条规则可选择：
  - **周期重复**：按天、小时、分钟组合设置间隔
  - **固定时间**：指定时间点，可按星期筛选（不选则每天执行）
- 关闭定时重启后，已设置的规则参数会保留，下次开启时自动恢复
- 定时重启与守护**相互独立**，可单独启用；与定时运行**互斥**

### 开启定时运行

- 点击列表行末的 **"开启定时运行"** 按钮，或右键选择 **"定时运行规则"**
- 与定时重启规则配置方式相同，但不会终止已有进程，仅在触发时启动程序
- 关闭定时运行后，已设置的规则参数会保留，下次开启时自动恢复
- 开启定时运行时，守护和定时重启会自动禁用

## 列表置顶

> **菜单路径：** 右键菜单 → 置顶 / 取消置顶

置顶功能让重要程序始终显示在列表最前面：

- 右键单击程序，选择 **"置顶"**，该程序将在程序图标前显示置顶图标并移到列表顶部
- 置顶图标自动适配主题：浅色模式使用深色图标，暗色模式使用浅色图标
- 支持多选后批量置顶/取消置顶
- 点击表头排序时，置顶的程序与未置顶的程序分别在各自区域内独立排序

---

## 启动延时

> **菜单路径：** 右键菜单 → 设置启动延时

当守护的程序自身带有更新机制或内置重启逻辑时（如热更新程序、自动升级器），程序退出后会在极短时间内自行重启。若守护进程在此期间也触发重启，可能导致程序被多次拉起，产生冲突。

**启动延时**功能解决了这一问题：在程序退出后，等待配置的秒数后再执行守护重启，给程序自身预留充足的重启窗口。

- 单位：秒（可设置 1 ~ 86400）
- 默认 1 秒
- 守护重启、定时重启均使用此延时；定时运行不使用此项

> ⚠️ 若程序稳定性较差，设置较长的启动延时可能导致程序中断期间短暂无法访问，请根据实际情况权衡设置。

## 失败重试

> **菜单路径：** 右键菜单 → 重试设置

当守护启动、定时重启或定时运行启动失败时，自动进入重试流程：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 重试间隔 | 30 秒 | 每次重试之间的等待时间 |
| 最大重试次数 | 10 次 | 0 = 无限制 |
| 最大重试时长 | 300 秒 | 0 = 无限制 |

重试成功后自动退出重试模式；重试耗尽后停止重试并可触发邮件通知。

## 邮件提醒

> **配置路径：** 配置 → 邮件提醒配置（SMTP 服务器设置）
>
> **开关路径：** 选项 → 邮件提醒（全局开关），或托盘右键菜单 → 邮件提醒

邮件提醒功能通过 SMTP 发送通知邮件，支持两级开关：

1. **全局开关**：菜单栏"选项 → 邮件提醒"或托盘右键菜单，默认**关闭**
2. **每程序开关**：右键菜单 → 邮件提醒设置，默认**关闭**

可配置的提醒事件：

| 事件 | 默认 | 说明 |
|------|------|------|
| 守护触发重启 | 关 | 守护检测到进程退出并重启时 |
| 启动失败 | 开 | 程序启动失败时 |
| 定时重启失败 | 开 | 定时重启后启动失败时 |
| 定时运行失败 | 开 | 定时运行启动失败时 |
| 进程退出 | 关 | 守护的进程退出时（延时重启前） |
| 重试耗尽 | 开 | 重试次数/时长耗尽时 |

> **去重机制：** 失败类事件（启动失败、定时重启失败、定时运行失败、重试耗尽）在下次成功之前只提醒一次，避免重复邮件轰炸。

## 自我守护

> **菜单路径：** 选项 → 自我守护，或托盘菜单 → 自我守护

开启后，SuperGuardian 会以看门狗模式启动自身的一个子进程（`--watchdog` 参数）持续监控主进程。若主进程意外崩溃，看门狗将自动将其重启（最多重试 5 次，失败后指数退避等待）。

**行为细节：**
- 通过菜单"退出"正常退出时，看门狗不会触发重启（写入 `self_guard_manual_exit` 标志）
- 可通过**测试 → 测试自我守护**强制结束主进程，验证恢复效果
- 看门狗启动/重启事件均记录到 `Logs/watchdog.log`

## 右键菜单

在程序列表上右键单击（支持多选后右键批量操作）：

| 菜单项 | 说明 |
|--------|------|
| 置顶 / 取消置顶 | 将程序置顶或取消置顶 |
| 备注 | 为程序设置备注名称，设置后在"程序"列显示备注（鼠标悬停仍显示原始名称） |
| 手动启动 | 立即启动选中程序（不影响守护状态） |
| 终止进程 | 结束选中程序的所有同名进程 |
| 设置启动程序/参数 | 修改启动的程序路径和启动参数 |
| 定时重启规则 | 设置定时重启规则（多条，周期/固定时间）；定时运行时不可选 |
| 设置启动延时 | 设置退出后延时重启的等待时长；定时运行时不可选 |
| 定时运行规则 | 设置定时运行规则（多条，周期/固定时间）；守护或定时重启开启时不可选 |
| 重试设置 | 配置启动失败后的重试策略 |
| 邮件提醒设置 | 配置该程序的邮件通知开关与事件 |
| 打开文件所在的位置 | 在资源管理器中显示程序文件（单选时显示） |
| 移除项 | 从列表中移除程序（需确认）；守护/定时重启/定时运行运行时不可选 |

> 不可选的菜单项以灰色显示，明确提示当前状态限制。

## 配置与日志

所有文件存储于程序所在目录，结构如下：

```
SuperGuardian.exe              # 主程序（单文件，无需安装）
img/
  app.ico                      # 程序图标（ICO 格式，用于 EXE 图标）
  light.png                    # 浅色主题切换图标
  dark.png                     # 暗色主题切换图标
  top_light.png                # 浅色置顶图标
  top_dark.png                 # 暗色置顶图标
Config/
  config.ini                   # 守护配置（程序列表、守护状态、选项等）
Logs/                          # 日志目录（首次写入时自动创建）
  watchdog.log                 # 看门狗事件日志（自我守护启动/重启记录）
  scheduled_restart.log        # 定时重启日志
  scheduled_run.log            # 定时运行日志
Cache/                         # 缓存目录（首次使用时自动创建）
bak/                           # 更新备份目录（更新时自动创建，按时间戳分子目录，最多保留 5 个）
```

配置文件格式为标准 INI，可手动编辑或通过**配置 → 导入/导出**迁移至其他机器。

## 从源码构建

### 环境准备

| 工具 | 版本 |
|------|------|
| Visual Studio | 2022 / 2026（需含 C++ 桌面开发负载） |
| MSVC 工具集 | v145（cl.exe 19.50+） |
| Qt | 6.10.2 静态构建（见下方说明） |
| Qt VS Tools | 最新版（VS 扩展） |

### 构建静态 Qt（仅首次需要）

项目提供了自动化脚本 `build_static_qt.bat`，一键完成 Qt 静态库的下载、编译与安装：

```bat
build_static_qt.bat
```

脚本将：
1. 从 `code.qt.io` 克隆 Qt 6.10.2 qtbase 源码
2. 使用 MSVC x64 配置静态构建（禁用 OpenGL、SQL、网络、DBus 等不需要的模块）
3. 编译并安装到 `C:\Qt\6.10.2\msvc2022_64_static`
4. 自动注册为 `Qt6.10.2_x64_static` 供 Qt VS Tools 识别

> 首次构建约需 **15 ~ 30 分钟**，需要约 **3 GB** 磁盘空间。

### 编译项目

用 Visual Studio 打开 `SuperGuardian.slnx`，选择 **Release | x64** 后生成。

或使用打包脚本（自动编译 + 压缩到 `package/` 目录）：

```powershell
.\package.ps1                   # 编译并打包，文件名包含时间戳
.\package.ps1 -Version 1.2.0   # 指定版本号
.\package.ps1 -SkipBuild        # 跳过编译，仅打包已有产物
```

**Release 配置特性：**
- 静态链接 Qt（`/MT`），输出单文件 `.exe`
- 无需 `windeployqt`，无外部 DLL 依赖
- 编译产物：`x64\Release\SuperGuardian.exe`（约 20 MB）
